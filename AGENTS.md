## Learned Workspace Facts

- **Backend stack:** **Flask** only — **`api.app`** (`python -m api`). Generic runbooks mentioning **FastAPI**, **`dev_stack.sh`**, **`/api/v2`**, or ports **8780/3780** refer to other projects; use **`docs/GETTING_STARTED.md`** and this repo’s scripts. Deprecated alternate API/UI prototypes (e.g. **`legacy/api/production_api.py`**, enhanced API variants, **`legacy/ui/quantum_portfolio_dashboard.jsx`**) live under **`legacy/`** — see **`legacy/README.md`**; not deployed, not CI-tested, not imported by the live app.
- Flask API: `python -m api` — documented default **http://localhost:5000** (health and OpenAPI under `/api/`). Package layout: **`api/app.py`** (Vercel: `pyproject.toml` **`app = "api.app:app"`**; **`vercel.json`** **`installCommand`** runs **`pip install -r deps/requirements-vercel.txt`** then **`pip install .`** so IBM/Qiskit packages are installed—root **`deps/requirements.txt`** alone is for CI/local); full **`deps/requirements.txt`** for local dev and CI and includes extras omitted on Vercel such as **networkx**, plotting, tests, and Braket/boto3; **`requires-python >=3.11`**).
- CORS: Flask uses **`CORS_ORIGINS`** (default **`http://localhost:3000`**). Browsers on other origins (e.g. Next on **3042** or **3001**) must list that origin in **`CORS_ORIGINS`**, or call the API same-origin via the Next **`/api/**` proxy / empty **`NEXT_PUBLIC_API_URL`** so CORS does not apply.
- **Fly.io two-app deploy** (Flask API + Next `web/`): **`docs/FLY_DEPLOY.md`** is the step-by-step guide. Two separate Fly apps: **API** (`deploy/docker/Dockerfile.fly`, `fly.toml`, gunicorn on port 5000, `api.app:app`) and **Next** (`web/Dockerfile`, `web/fly.toml`, standalone output, port 3000). **Web** from repo root: **`fly deploy ./web --config fly.toml`** (build context must be `web/`). **`API_PROXY_TARGET`** is **`fly secrets`** on the web app (runtime proxy in `web/src/app/api/[[...path]]/route.ts`). **`NEXT_PUBLIC_API_KEY`** is a **Docker `--build-arg`** (client bundle). See **`docs/FLY_DEPLOY.md`** if logs show **`ECONNREFUSED 127.0.0.1:5000`**. **API** container should **`gunicorn --bind 0.0.0.0:5000`** to match **`fly.toml`** **`internal_port`**; shell-form **`CMD`** that uses **`$$PORT`** is wrong (**`$$`** is the shell PID, not **`$PORT`**). Next server calls Flask via Fly private DNS: **`API_PROXY_TARGET=http://<api-app-name>.internal:5000`** (set via `fly secrets set` on the web app; never commit). `NEXT_PUBLIC_API_KEY` must match `API_KEY` on the API app. SQLite persistence requires a Fly volume and `API_DB_PATH=/data/api.sqlite3`.
- Vercel **two-project** deploy (API + Next): **`docs/VERCEL_TWO_PROJECTS.md`**, wiring checklist **`docs/VERCEL_WIRE_NEXT_API.md`**. Python at repo root vs Next **Root Directory** **`web`** — use **`vercel link`** separately in each folder so API env (**`API_KEY`**, optional **`CORS_ORIGINS`**, optional **`DASHBOARD_PUBLIC_URL`**) and Next env stay on the right project. Multiple Git remotes (e.g. **`origin`** vs **`quantum`**) may point to different GitHub repos; Vercel builds the connected repo—push the branch there, not only to a fork, if they differ. **Option 1 (direct):** **`NEXT_PUBLIC_API_URL`** = API base + **`CORS_ORIGINS`** on the API includes the Next origin. **Option 2 (proxy, often simpler):** leave **`NEXT_PUBLIC_API_URL`** unset; on the **Next** deployment set **`API_PROXY_TARGET`** to the **public HTTPS** API base (no trailing slash) — **not** `http://127.0.0.1:5000` (local-only; breaks on Vercel). **`NEXT_PUBLIC_API_KEY`** must match **`API_KEY`** on the API. CLI / scripts: **`docs/VERCEL_CLI.md`**, **`scripts/vercel-deploy-api.sh`**, **`scripts/vercel-deploy-web.sh`**, Run Task → Vercel (`.vscode/tasks.json`). Consolidated stack/deps overview: **`docs/HOSTING_NEXT_AND_FLASK_ARCHITECTURE.md`**.
- CRA dashboard: `cd frontend && npm install && npm start` — **http://localhost:3000**; `frontend/package.json` proxies API requests to the Flask port. Lab UI: **`frontend/src/CustomizableQuantumDashboard.js`**. **Hugging Face Space / Docker HF:** **`deploy/docker/Dockerfile.hf`** builds **`frontend/`** and runs **`serve_hf.py`**; README **`app_port: 7860`** — **not** the Next **`web/`** app. Incomplete **`web/`** checkouts (e.g. only **`web/.next`**) — treat as a separate stack from HF/CRA until source matches **`hf-deploy`**. **Gradio lite** (same Flask API as **`web/`**): **`scripts/gradio_lite/`**, **`scripts/gradio_portfolio_demo.py`**, **`deps/requirements-gradio.txt`** — lite Portfolio Lab / Quantum Engine–style UI; default Gradio port **7861** (API on **5000**).
- Next.js UI: **`web/`** — use **`scripts/run-next-web.sh`** to run `next dev` with **`NEXT_WEB_PORT`** (default **3042**) so it stays clear of CRA **3000** and Flask **5000**; requires **`cd web && npm install`** when dependencies are not yet present. **`/api/*`** to Flask: request-time **`web/src/app/api/[[...path]]/route.ts`** using **`API_PROXY_TARGET`** (not **`next.config`** rewrites — those bake at **`next build`**; see **`web/next.config.ts`**). Full **`web/`** App Router source is tracked on branch **`hf-deploy`**; if **`web/package.json`** is missing, restore with **`git checkout hf-deploy -- web`** (other branches may omit it).
- **Next dev lock + web audit artifacts:** Next 16 refuses a second **`next dev`** on the same **`web/`** checkout (“Another next dev server is already running”). Smokes for request-time **`/api/**`** proxy on **:3042** alongside a running :3000 dev use **`NEXT_PUBLIC_API_URL="" npm run build`** then **`next start -p 3042`** under **`web/`** (or stop the :3000 dev). Operator checklist: **`docs/AUDIT_REMAINING_WORK.md`**; machine-readable + narrative audit: **`docs/page-audit.json`** (e.g. **`manual_flows[]`**, **`audited_ports`**, per-check **`port`**) and **`docs/PAGE_AUDIT.md`**.
- Flask app key (**`API_KEY`** / **`NEXT_PUBLIC_API_KEY`** / **`X-API-Key`**) is separate from the **IBM Quantum** API token (stored per tenant via the API database), and from CORS.
- IBM / integration persistence: tenant-scoped secrets (e.g. IBM token) live in the API SQLite DB (**`API_DB_PATH`**, default under **`data/`**); new machines or empty DBs need credentials re-entered or the DB/volume restored alongside **`API_KEY`** env alignment.
- IBM Quantum: optional **instance CRN** is stored in tenant integration metadata and passed to **`QiskitRuntimeService(..., instance=...)`** when set; **`POST /api/config/ibm-quantum/verify`** validates token (and optional instance) without persisting; see **`docs/ibm-quantum-credentials.md`**. **Vercel** API installs **`qiskit`** / **`qiskit-ibm-runtime`** from **`deps/requirements-vercel.txt`** (pins align with **`deps/requirements-ibm-quantum.txt`**); Python serverless functions have an **~245 MB uncompressed** bundle cap (build errors comparing total size to that limit). Shrink upload with **`.vercelignore`** (commit it so CI/Vercel see it) and **`vercel.json`** **`excludeFiles`**. If **`site-packages`** after install still exceeds the cap, use a **non-serverless** host or trim deps. Changing those Python deps requires redeploying the **Flask API** Vercel project (not the Next project). A live **`qiskit-ibm-runtime` ImportError** often means the API bundle is stale (build failed or GitHub checks blocked deploy), not a Next install issue. **Smoke test:** **`POST /api/config/ibm-quantum/smoke-test`** runs **`hardware_smoke_test`** in **`services/ibm_quantum.py`** — VQE-shaped single shot (**EfficientSU2** + **`load_market_payload`**), default tickers **Mag 7 + JPM** (optional body overrides), **`hardware`** or **`simulator`** mode; **`notebooks/test.ipynb`** mirrors this path.
- IBM Quantum **tenant resolution** (`integration_effective_tenant_id` in **`api/app.py`**): IBM tokens are stored **per `tenant_id`** in the API DB. With the **shared** **`API_KEY`** / **`NEXT_PUBLIC_API_KEY`**, if the browser does not send **`X-Tenant-Id`** (Next only adds it when **`ql_active_tenant`** is set in **`localStorage`** — see **`web/src/lib/api.ts`**), the backend often resolves **`tenant_id`** to **`"default"`**, so **one shared IBM integration namespace** applies to all such clients. Giving each visitor their own IBM credentials in production requires **server-trusted identity** (e.g. per-user API keys in **`api_keys`**, auth, or another mapping)—not only uploading a token while still hitting **`default`**.
- **`qaoa`** / **`hybrid_qaoa`** portfolio objectives: IBM Quantum Runtime path (Estimator-style) over the same QUBO family as **`qubo_sa`**, with classical fallback when no token; **`api/app.py`** groups **`vqe`**, **`qaoa`**, and **`hybrid_qaoa`** as IBM Runtime objectives. Implementation: **`methods/qaoa.py`**, **`methods/hybrid_pipeline.hybrid_qaoa_weights`**, **`core/optimizers/qaoa.py`**, **`core/optimizers/hybrid_qaoa.py`**.
- Market data: **`DATA_PROVIDER`** env selects the provider; default is **`tiingo`** when **`TIINGO_API_KEY`** is set, otherwise falls back to **`yfinance`** (legacy, deprecated). Primary implementation is in **`services/data_provider_v2.py`** (`TiingoProvider`, `MarketDataProvider`, `fetch_market_data`, `fetch_price_panel`). `services/market_data.fetch_market_data` delegates to v2. `services/backtest.py` uses `fetch_price_panel` (no direct yfinance). **`TIINGO_API_KEY` for local dev is read with the Flask/API process** — set it in **repo root `.env.local`** (or the API host’s env), not only in **`web/.env.local`**. yfinance emits a `DeprecationWarning` at runtime when selected. Optional **`METADATA_SECTOR_SOURCE=yfinance`**: when using Tiingo for prices, merge **only** sector/industry from yfinance via **`services/market_data.get_yfinance_sector_industry`** (extra HTTP; omit or leave unset to keep Tiingo placeholders).
- **Automated QA (mirror CI):** **`.github/workflows/ci.yml`** — Python **3.11** and **3.12**: **`pip install -r deps/requirements.txt`** + **`pip install -e .`**, then **`python -m pytest tests/ -v --cov=core --cov=services --cov=api --cov-report=term-missing --tb=short`**; **`tests/conftest.py`** sets **`collect_ignore`** for **`test_enhanced_system.py`** (import chain requires missing **`config.qsw_config`**) so default collection does not error. **`web/`**: **`npm ci`**, **`npm run lint`**, **`npm run build`**; legacy **`frontend/`**: **`npm ci`**, **`CI=true npm test -- --watchAll=false`**, **`npm run build`**. **`docker build -f deploy/docker/Dockerfile`** for API + **`frontend/`** images runs on **`push` to `main`** only (after those jobs pass).
- **PDF report generation**: `GET /api/export/report/<run_id>.pdf` — implemented in **`services/report_generator.py`** (Jinja2 + WeasyPrint HTML→PDF). Jinja2 templates live in **`templates/`** at repo root (`report.html`, `report.css`). WeasyPrint requires system libs (`libpango-1.0-0`, `libpangocairo-1.0-0`, `libcairo2`, `libgdk-pixbuf2.0-0`) — already added to **`deploy/docker/Dockerfile.fly`**. **WeasyPrint cannot run on Vercel serverless** (system deps unavailable); the PDF endpoint is Fly.io/local-only. Run data sourced via `lab_run_service.get_run(run_id, tenant_id)` — not any `run_repository` alias. Frontend download uses fetch+Blob+`createObjectURL` pattern (not a plain `<a href>`) because the endpoint requires `X-API-Key` header (`downloadReportPdf` in **`web/src/lib/api.ts`**).
